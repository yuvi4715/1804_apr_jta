DIRECTIVES

Directives are used to provide instruction within the DOM.
They arguably proivde directions for angular within the view of the application. (The html) All directives are provided by the angular library as well as offering the developer a means to create their own custom directives.

*ngIf

We can use *ngIf to determine whether or not we should show and element. It serves as a conditional where if it's true, we show the element, else we hide it.

Why do some Directives have an '*' before them, and others don't?


The * is simply syntactic sugar that points out that the directive in question can be used to change the structure of the html page. e.g. removing or adding elements.

ngStyle

ngStyle is a directive that is used for applying and object that represents the css of an element. By utilizing this, we can dynamically style any tag in our html.

*ngFor

*ngFor is a tag that can add elements from a list. We can use it to duplicate whatever element it is in, for each element in the provided list. (Note: This is a structural directive, just like *ngIf)
